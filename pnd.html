Practical Network Defense

The following page contain the exercises done during the course using Kathará: a container-based framework for experimenting computer networking.

Important tips for kathara:

  • When you create a folder with pc name (e.g. pc1), every time you start kathará, the content inside pc1 folder will be copied inside the root directory of pc1. This is good when for example you don't want to write inside /etc/network/interfaces every time at startup;
  • If you don't want to write every time the commands in the terminal, you just need to put them inside 'startup' file (e.g. pc1.startup);
  • run kathara with '--noterminals' option to start the device without opening the 'ugly' terminals:
    • then use 'kathara connect pcN' to open a terminal.

Main commands (aliases):

  • start (restart) a lab: 'lstart/lrestart'
  • stop a running lab: 'lclean'
  • wipe the kathara environment: 'kwipe'
  • list virtual networks and virtual interfaces of VMs: 'docker network list'
  • clean all the IP addresses of an ethernet interface: 'ip addr flush eth0'
  • 'arp/ip neigh'
  • show all the ip addresses that we have: 'ifconfig/ip addr'
  • show the routes: 'ip route'
